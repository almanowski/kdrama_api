Documentation

Objective

To build the server-side component of a "Korean drama" web application. The web application will provide users with access to information about different dramas, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favourite Korean dramas.

Dramas

REQUEST URL HTTP METHOD REQUEST DATA FORMAT RESPONSE DATA BODY FORMAT
Get a list of Korean dramas /korean-dramas GET None A JASON object holding data about all korean dramas
Get data about a single drama /korean-dramas/[title] GET None A JASON object holding data about a single drama, containing a title, epsiodes, release Year, description, genre, director, image URL

Get data about a genre by name/title (e.g., "Thriller") /genres/[genre] GET None A text message, indicating a drama genre list
Get data about a director by name /directors/[name] GET None A text message, indicating a certain drama director

Users

REQUEST URL HTTP METHOD REQUEST DATA FORMAT RESPONSE DATA BODY FORMAT
Allow new users to register /users POST None A text message, indicating the user was registered
Allow users to update their user info (username) /users/[id] PUT None A text message, indicating the username was updated
Allow users to add a drama to their list of fav /users/[id]/favs/[dramaId] PUT None A text message, indicating that the drama has been added
Allow users to remove a drama from their list of favourites /users/[id]/favs/[dramaId] DELETE None A text message, indicating that the drama has been removed
Allow existing users to deregister /users/[id] DELETE None A text message, indicating that the email has been removed
